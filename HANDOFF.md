# HANDOFF.md — Inkwell

A handoff briefing for the next AI agent picking up this project cold. Read this first; then read [`AGENTS.md`](AGENTS.md) for the full architecture and the render-pipeline contract before you edit code.

## TL;DR

Inkwell is a **single-file, zero-build, browser-based Markdown editor** with a live side-by-side preview. Everything — HTML, CSS, JS — is in one file: **`markdown-editor.html`** (~1,625 lines). Open it in a browser and it runs. No `package.json`, no bundler, no framework, no server. It targets running as a **Claude.ai artifact**, which is the source of most of its unusual constraints (see "Sharp edges").

It is also the behavioral reference for a planned **SwiftUI macOS port** (not yet started in this repo).

## What it does (feature inventory)

- Live Markdown → HTML preview (marked + DOMPurify), updated on every keystroke via a single `render()` call.
- Toolbar formatting (bold/italic/strike/headings/lists/quote/code/link/image/table/diagram/HR) + keyboard shortcuts (⌘B/I/K, ⌘Z/⌘⇧Z).
- Custom undo/redo stack (the textarea's native undo is bypassed because programmatic edits break it).
- Smart lists: Enter continues/exits a list, Tab/Shift-Tab indents.
- Find & replace (⌘F), textarea-based — one selection visible at a time (no all-match highlight; known limitation).
- Outline/TOC sidebar from ATX headings, click to jump editor + preview.
- Syntax highlighting (highlight.js) with per-block Copy buttons.
- **Mermaid** diagrams (all 17 types, via a helper modal with templates).
- **KaTeX** math (`$…$`, `$$…$$`).
- **Custom ` ```roadmap ` block** — a bespoke (non-Mermaid) Agile swimlane renderer. See `AGENTS.md` → "Custom syntaxes" for its directive grammar.
- YAML front-matter → rendered metadata card.
- Sortable tables (preview-only; does not rewrite the source).
- Interactive task checkboxes (these *do* rewrite the source so they persist + undo).
- Themes (light/dark, token-based CSS variables; accent is ink-teal).
- Import (button + drag-drop), export (.md / standalone .html / copy HTML / print-to-PDF).
- Autosave + document name, persisted via the artifact `window.storage` KV store.
- About modal.

## How to run, test, and verify a change

- **Run:** open `markdown-editor.html` in any modern browser. To exercise autosave/`window.storage`, it must run inside the Claude.ai artifact sandbox; locally `window.storage` is absent and the app degrades to in-memory (footer shows "In-memory only").
- **Edit:** change the one file directly. No compile step.
- **Syntax-check the JS** without a browser:
  ```bash
  python3 -c "import re;open('/tmp/m.js','w').write(re.findall(r'<script>(.*?)</script>',open('markdown-editor.html').read(),re.S)[-1])"
  node --check /tmp/m.js
  ```
  (`node --check` validates syntax only — the code uses browser globals it can't execute.)
- **Manual test matrix** after any change: typing + live preview; undo/redo; find/replace; outline jump; each export path; theme toggle; a Mermaid diagram; a KaTeX expression; a ` ```roadmap ` block; a sortable table; a task checkbox; import a file; refresh to confirm autosave restore.

## Sharp edges — read before you touch code

These are deliberate, not bugs. Don't "fix" them away.

1. **No `localStorage` / `sessionStorage`.** Blocked in the artifact sandbox. Use `window.storage` (async `get`/`set`/`delete`/`list`), feature-detected via `hasStore`. Degrade to in-memory if absent.
2. **No native `alert` / `confirm` / `prompt`.** The sandbox silently suppresses them. Every dialog is a custom in-page `.modal-backdrop`. Don't reintroduce native dialogs.
3. **No real filesystem paths.** The File API exposes only `file.name`. The header shows the name only — never fabricate a path (including the browser's `C:\fakepath\`).
4. **The render pipeline order is a contract.** KaTeX/Mermaid/roadmap HTML is injected **after** DOMPurify (sanitizing would mangle them). This is safe *only* because each is generated by a trusted library from the user's own input (Mermaid strict mode; roadmap text is `esc()`-escaped). Anything you inject as raw HTML must come from a trusted generator, never from unsanitized user paste. See the numbered pipeline in `AGENTS.md`.
5. **Math is protected with placeholder tokens** (`n`) around marked, so markdown can't turn `$x_i$` into `$x<em>i</em>$`. If you change the pipeline, preserve `protectMath`/`restoreMath`.
6. **Every CDN library is feature-detected** (`if (!window.mermaid) return;` etc.). If a CDN fails, that feature disables and the editor keeps working. Never assume a library is present.
7. **Mermaid leaves orphan nodes on `<body>`.** `mermaidSandbox` (off-screen but *real width* — gantt/xy/timeline collapse at zero width) and `sweepMermaidOrphans` exist to contain and clean these. Don't remove them.
8. **Two different table behaviors:** sortable-table sorting is **preview-only** and resets on re-render; task-checkbox toggling **rewrites the Markdown source**. Don't conflate them.
9. **Dead code to know about:** the `SAMPLE` constant (line ~678) is defined but **not used at boot** — `init()` loads saved content or leaves the editor empty with a placeholder. If you want a first-run sample document, wire `SAMPLE` into `init()`; don't assume it's already shown.

## Conventions

- 2-space indent, semicolons, single quotes, vanilla DOM APIs (no jQuery/framework). Small named functions grouped under `/* ---- section ---- */` comments.
- All colors come from CSS custom properties under `:root` / `[data-theme="dark"]`. Add UI with these tokens, not hard-coded hex.
- Dependencies (all cdnjs, in `<head>`): marked 12.0.2, DOMPurify 3.1.6, highlight.js 11.9.0, mermaid 10.9.1, KaTeX 0.16.9.

## Common tasks (quick index — full detail in `AGENTS.md`)

- **Add a toolbar button:** add `<button class="tool" data-act="x">` in `#toolbar`, add `x: () => …` to the `actions` map. The toolbar handler wraps it in history `flush()`/`commitNow()`.
- **Add a render step:** write the function, call it inside `render()` after `preview.innerHTML` is set, and feature-detect any library.
- **Add a code-block language renderer:** add `|| block.classList.contains('language-X')` to the `highlightCode` skip check, then handle `pre > code.language-X` in your own renderer (this is exactly how `roadmap` works).
- **Add a modal:** copy an existing `.modal-backdrop`, wire open/close, and add it to the Escape chain in the global keydown handler.

## What's next / where this is headed

- **SwiftUI macOS port.** Several behaviors here are web-sandbox workarounds that the native app should replace with platform primitives rather than port literally:
  - custom undo stack → `NSTextView` `UndoManager`
  - `window.storage` + filename label → `DocumentGroup` / `FileDocument` with a real file URL
  - custom modals → native sheets/panels
  - Mermaid/KaTeX/highlight.js reliance is the main argument for a `WKWebView` render path over a pure-SwiftUI renderer.
- If you change feature behavior in `markdown-editor.html`, note whether the (separate) native build prompt needs updating to match.

## Things this AI should NOT do without asking

- Don't add a build step, framework, bundler, or npm dependency — the single-file constraint is the whole point.
- Don't introduce browser storage or native dialogs (see Sharp edges).
- Don't commit or push unless the user asks; if on `main`, branch first.
